
# Не заблокированы ошибки зависимостей подключенных проектов

|||
|:-|:-|
| Id | **SSDT0003**
| Мнемо | DACPAC_WARNING_SUPPRESSED
| Серьёзность | ⚠ Предупреждение
| Категория | [Проект](./Группа_Проект.md)
| Исходный код | [DacpacWarningSuppressedRule.cs](../../../Rules/SqlProject/DacpacWarningSuppressedRule.cs)

## Причина

Правило срабатывает, если для подключенного dacpac-файлом проекта не выставлено свойство, включающее подавление ошибок отсутствующих зависимостей.
<p id="descr">Задайте свойство SuppressMissingDependenciesErrors.</p>

## Примеры

Некорректно

```xml
<ArtifactReference Include="..\..\_DACPACs\frontend.dacpac">
  <HintPath>..\..\_DACPACs\frontend.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>False</SuppressMissingDependenciesErrors> <!-- here -->
  <DatabaseSqlCmdVariable>frontend</DatabaseSqlCmdVariable>
</ArtifactReference>
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">  <!-- here -->
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```

Корректно

```xml
<ArtifactReference Include="..\..\_DACPACs\frontend.dacpac">
  <HintPath>..\..\_DACPACs\frontend.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>  <!-- here -->
  <DatabaseSqlCmdVariable>frontend</DatabaseSqlCmdVariable>
</ArtifactReference>
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>  <!-- here -->
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```

## Подсказки

💡 При добавлении ссылки на другой проект через dacpac, нужно в диалоге добавления ссылки включить опцию "Suppress errors caused by unresolved references in the referenced project".
