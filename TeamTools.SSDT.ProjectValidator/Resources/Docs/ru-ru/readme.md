## Как внести вклад

Вы можете улучшать документацию, редактируя маркдаун-файлы. Также приветствуется помощь в переводе документации на другие языки.
Все изменения должны проходить через пулл-реквесты и ревью .

Для файла описания конкретного правила используйте [этот шаблон](./_rule_template.md), при этом

- Старайтесь сохранять разметку файлов документации настолько простой, насколько это возможно
- Если на момент составления описания правила нет возможности добавить подсказки или примеры - не добавляйте эти блоки совсем, их всегда можно добавить позднее
- Все фрагменты внутри фигурных скобок в шаблоне должны быть заменены актуальным текстом (уже без фигурных скобок)
- Не размещайте HTML- или маркдаун-разметку внутри элемента `<p id="descr">` - содержимое этого элемента также
  отображается во всплывающей подсказке о нарушении правила в интерфейсе Visual Studio, и эта подсказка поддерживает только простой текст
- Для корректного отображения разметки в Bitbucket перед самым первым заголовком обязательно должна быть хотя бы одна пустая строка
- При изменении основного сообщения о нарушении правила соответствующая строка в файле [ViolationMessages](../../../Resources/ViolationMessages.ru-ru.json), а также на страницах-перечнях состава групп и категорий правил должна быть изменена аналогично
- Чтобы разобраться, как оформлять файлы с документацией правил, проще всего посмотреть уже имеющиеся примеры.
