
# Дублирующееся включение в проект

|||
|:-|:-|
| Id | **SSDT0001**
| Мнемо | PROJECT_DUP_INCLUDE
| Серьёзность | ⚠ Предупреждение
| Категория | [Проект](./Группа_Проект.md)
| Исходный код | [DuplicateIncludeRule.cs](../../../Rules/SqlProject/DuplicateIncludeRule.cs)

## Причина

Правило срабатывает, если один и тот же объект включен в проект несколько раз. Проверяются файлы и подключенные проекты.
<p id="descr">Удалите дублирующуюся запись добавления в проект.</p>

## Примеры

Некорректно

```xml
<Build Include="Stored Procedures\dbo.proc_a.sql" />
<Build Include="Stored Procedures\dbo.proc_b.sql" />
<Build Include="Stored Procedures\dbo.proc_a.sql" />
<Build Include="Stored Procedures\dbo.proc_c.sql" /> <!-- here -->
<!-- ... -->
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">  <!-- here -->
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```

Корректно

```xml
<Build Include="Stored Procedures\dbo.proc_a.sql" />
<Build Include="Stored Procedures\dbo.proc_b.sql" />
<Build Include="Stored Procedures\dbo.proc_c.sql" />
<!-- ... -->
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```
