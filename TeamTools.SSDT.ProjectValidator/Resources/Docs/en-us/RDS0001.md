
# The server name is not fully qualified

|||
|:-|:-|
| Id | **RDS0001**
| Mnemo | SERVER_LINK_FULLY_QUALIFIED
| Severity | ⚠ Warning
| Category | [Datasource](./Group_Datasource.md)
| Source code | [ServerNameFullyQualifiedRule.cs](../../../Rules/ReportDatasource/ServerNameFullyQualifiedRule.cs)

## Cause

This rule is triggered if the database server is not specified with its fully qualified domain name (FQDN).
<p id="descr">Specify the fully qualified domain name of the server.</p>

## Examples

Bad

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="srv.myDb">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv;Initial Catalog=myDb;Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```

Good

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="srv.myDb">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv.domain.org;Initial Catalog=myDb;Integrated Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```

## Tips

💡 FQDN (Fully Qualified Domain Name) of the server is the name of the following format: server-name.domain-name.zone. For example, my_srv.my-domain.local
