
# Variable defined in sqlproj but missing from publish config

|||
|:-|:-|
| Id | **SSDT0010**
| Mnemo | PUBLISH_XML_VARIABLE_MISSING
| Severity | ⚠ Warning
| Category | [Project](./Group_Project.md)
| Source code | [PublishXmlHasSameValuesListRule.cs](../../../Rules/SqlProject/PublishXmlHasSameValuesListRule.cs)

## Cause

This rule is triggered if a variable declared in the project file is missing from the publish configuration file.
<p id="descr">Add the variable to the publish.xml file.</p>

## Examples

Bad

```xml
<!-- sqlproj fragment -->
  <ArtifactReference Include="..\..\_DACPACs\another_project.dacpac"><!-- ... -->
  <ArtifactReference Include="..\..\_DACPACs\my_project.dacpac"><!-- ... -->
<!-- ... -->
  <SqlCmdVariable Include="another_project"><!-- ... -->
  <SqlCmdVariable Include="my_project"><!-- ... -->

<!-- publish.xml fragment -->
<ItemGroup>
    <SqlCmdVariable Include="my_project"><!-- ... -->
</ItemGroup>
```

Good

```xml
<!-- sqlproj fragment -->
  <ArtifactReference Include="..\..\_DACPACs\another_project.dacpac"><!-- ... -->
  <ArtifactReference Include="..\..\_DACPACs\my_project.dacpac"><!-- ... -->
<!-- ... -->
  <SqlCmdVariable Include="another_project"><!-- ... -->
  <SqlCmdVariable Include="my_project"><!-- ... -->

<!-- publish.xml fragment -->
<ItemGroup>
  <SqlCmdVariable Include="another_project"><!-- ... -->
  <SqlCmdVariable Include="my_project"><!-- ... -->
</ItemGroup>
```
