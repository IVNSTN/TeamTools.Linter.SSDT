
# Dacpac reference warnings not suppressed

|||
|:-|:-|
| Id | **SSDT0003**
| Mnemo | DACPAC_WARNING_SUPPRESSED
| Severity | ⚠ Warning
| Category | [Project](./Group_Project.md)
| Source code | [DacpacWarningSuppressedRule.cs](../../../Rules/SqlProject/DacpacWarningSuppressedRule.cs)

## Cause

This rule is triggered if the referenced dacpac does not have the SuppressMissingDependenciesErrors property set to True.
<p id="descr">Set the SuppressMissingDependenciesErrors property.</p>

## Examples

Bad

```xml
<ArtifactReference Include="..\..\_DACPACs\frontend.dacpac">
  <HintPath>..\..\_DACPACs\frontend.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>False</SuppressMissingDependenciesErrors> <!-- here -->
  <DatabaseSqlCmdVariable>frontend</DatabaseSqlCmdVariable>
</ArtifactReference>
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">  <!-- here -->
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```

Good

```xml
<ArtifactReference Include="..\..\_DACPACs\frontend.dacpac">
  <HintPath>..\..\_DACPACs\frontend.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>  <!-- here -->
  <DatabaseSqlCmdVariable>frontend</DatabaseSqlCmdVariable>
</ArtifactReference>
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>  <!-- here -->
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```

## Tips

💡 When adding a reference to another project, you need to check the checkbox "Suppress errors caused by unresolved references in the referenced project" in the "Add Database Reference" dialog.
