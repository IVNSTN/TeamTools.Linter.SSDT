
# DataSource name contains illegal symbols

|||
|:-|:-|
| Id | **RDS0005**
| Mnemo | DATASOURCE_NAME_ALLOWED_SYMBOLS
| Severity | ⚠ Warning
| Category | [Datasource](./Group_Datasource.md)
| Source code | [DatasourceNameAlphanumericRule.cs](../../../Rules/ReportDatasource/DatasourceNameAlphanumericRule.cs)

## Cause

This rule is triggered if a DataSource name contains illegal symbols.
<p id="descr">Remove all illegal symbols from the DataSource name.</p>

## Examples

Bad

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="$Ё">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv.domain.org;Initial Catalog=myDb;Integrated Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```

Good

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="srv.myDb">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv.domain.org;Initial Catalog=myDb;Integrated Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```

## Tips

💡 It is prohibited to use any symbols other than Latin letters, dashes (-), underscores (_) or periods (.).
