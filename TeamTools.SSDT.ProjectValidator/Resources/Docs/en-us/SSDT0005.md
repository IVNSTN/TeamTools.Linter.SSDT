
# Dacpac reference CopyLocal/Private option must not be True

|||
|:-|:-|
| Id | **SSDT0005**
| Mnemo | DACPAC_COPY_LOCAL
| Severity | ⚠ Warning
| Category | [Project](./Group_Project.md)
| Source code | [DacpacCopyLocalRule.cs](../../../Rules/SqlProject/DacpacCopyLocalRule.cs)

## Cause

This rule is triggered if the CopyLocal/Private option of the referenced dacpac file is set to True explicitly.
<p id="descr">Remove the explicit True setting for CopyLocal/Private.</p>

## Examples

Bad

```xml
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
  <Private>True</Private>
</ArtifactReference>
```

Good

```xml
<ArtifactReference Include="..\..\_DACPACs\another_project.dacpac">
  <HintPath>..\..\_DACPACs\another_project.dacpac</HintPath>
  <SuppressMissingDependenciesErrors>True</SuppressMissingDependenciesErrors>
  <DatabaseSqlCmdVariable>another_project</DatabaseSqlCmdVariable>
</ArtifactReference>
```
