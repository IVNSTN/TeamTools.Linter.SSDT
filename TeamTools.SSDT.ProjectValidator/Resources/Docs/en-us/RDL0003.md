
# Defining a connection string in a report

|||
|:-|:-|
| Id | **RDL0003**
| Mnemo | CONNECTION_STRING_ILLEGAL
| Severity | ⚠ Warning
| Category | [Report](./Group_Report.md)
| Source code | [ConnectionStringInReportRule.cs](../../../Rules/Reports/ConnectionStringInReportRule.cs)

## Cause

This rule is triggered if a connection string definition is found in the report.
<p id="descr">Remove the connection string from the report.</p>

## Examples

Bad

```xml
<?xml version="1.0" encoding="utf-8"?>
<Report MustUnderstand="df" xmlns="...">
  <DataSources>
    <DataSource Name="TEST_DS">
      <ConnectionProperties>
        <DataProvider>SQL</DataProvider>
        <ConnectString>Data Source=srv.domain.org;Initial Catalog=reporting_database</ConnectString>
        <IntegratedSecurity>true</IntegratedSecurity>
      </ConnectionProperties>
    </DataSource>
  </DataSources>
</Report>
```

Good

```xml
<?xml version="1.0" encoding="utf-8"?>
<Report MustUnderstand="df" xmlns="...">
  <DataSources>
    <DataSource Name="TEST_DS">
      <ConnectionProperties>
        <DataProvider>SQL</DataProvider>
      </ConnectionProperties>
    </DataSource>
  </DataSources>
</Report>
```
