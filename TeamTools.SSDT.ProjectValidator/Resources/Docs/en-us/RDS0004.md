
# DataSource name is too short

|||
|:-|:-|
| Id | **RDS0004**
| Mnemo | DATASOURCE_NAME_LENGTH
| Severity | ⚠ Warning
| Category | [Datasource](./Group_Datasource.md)
| Source code | [DatasourceNameMinLengthRule.cs](../../../Rules/ReportDatasource/DatasourceNameMinLengthRule.cs)

## Cause

This rule is triggered if the length of a DataSource's name is 1.
<p id="descr">Give the DataSource a more meaningful name.</p>

## Examples

Bad

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="A">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv.domain.org;Initial Catalog=myDb;Integrated Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```

Good

```xml
<?xml version="1.0" encoding="utf-8"?>
<RptDataSource Name="srv.myDb">
  <ConnectionProperties>
    <Extension>SQL</Extension>
    <ConnectString>Data Source=srv.domain.org;Initial Catalog=myDb;Integrated Security=SSPI</ConnectString>
  </ConnectionProperties>
</RptDataSource>
```
